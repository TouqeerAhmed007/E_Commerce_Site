------------------------------------------------------------
8. STEP-BY-STEP IMPLEMENTATION PLAN
------------------------------------------------------------

STEP 1: Project Setup
  - Initialize project: npm init -y
  - Install dependencies:
    npm install express mongoose bcryptjs jsonwebtoken
    dotenv cors helmet express-validator express-rate-limit morgan
  - Install dev dependency: npm install -D nodemon

STEP 2: Database Connection
  - Set up MongoDB (local or MongoDB Atlas free tier)
  - Create config/db.js to connect via Mongoose

STEP 3: Build Models
  - Create User, Product, Order, Category schemas/models
  - Hash passwords using a pre-save hook in the User model

STEP 4: Authentication System
  - Build register endpoint (validate + hash password + save)
  - Build login endpoint (compare password + issue JWT)
  - Build authMiddleware.js to verify JWT on protected routes

STEP 5: RBAC Middleware
  - Build roleMiddleware.js -> authorizeRole(...roles)
  - Apply to admin-only routes (product CRUD, order status
    update, dashboard)

STEP 6: Product Management (Admin)
  - Build CRUD endpoints for products
  - Protect with authMiddleware + authorizeRole("admin")

STEP 7: Customer Order Flow
  - Build "place order" endpoint (deduct stock, create order
    record with status "pending")
  - Build "view my orders" endpoint (filter by customerId
    from JWT)

STEP 8: Admin Order Management
  - Build "view all orders" endpoint - sort by status="pending"
    first, then by createdAt descending, so newest pending
    requests show at the top
  - Build "update order status" endpoint

STEP 9: Admin Dashboard
  - Aggregate stats: total orders, total revenue, low stock
    items, pending order count

STEP 10: Frontend Integration
  - Build simple HTML/CSS/JS pages (or React):
      * index.html / shop.html  -> product listing (customer)
      * login.html / register.html
      * cart.html / checkout.html
      * admin.html -> dashboard + pending orders + product
        management forms
  - Use fetch() to call your Express APIs
  - Store JWT in memory or httpOnly cookie (preferred over
    localStorage for security)

STEP 11: Testing
  - Test all endpoints with Postman
  - Test RBAC: ensure customer cannot access admin routes
    (expect 403)
  - Test validation: try weak passwords, invalid emails,
    duplicate registrations

STEP 12: Security Hardening Pass
  - Apply helmet, rate limiting on auth routes
  - Review for injection vulnerabilities
  - Ensure error messages don't leak sensitive info
  - Add .env to .gitignore (never commit secrets)
